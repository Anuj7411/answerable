# How to fix a missing canonical URL

**Problem.** Audit check **A4** flagged this page as missing a canonical URL. Without it, near-duplicate variants of the same page (trailing slashes, query strings, locale paths) compete for the same ranking signal and dilute each other.

**Solution.** Declare `<link rel="canonical" href="..."> ` in `<head>` pointing at the single URL you want search engines to consolidate signals onto.

## Recommended way: `defineSeo()`

If you're using `@answerable/metadata`, the canonical is one field:

```tsx filename="app/layout.tsx"
import { defineSeo } from '@answerable/metadata';
import type { Metadata } from 'next';

export const metadata: Metadata = defineSeo({
  title: 'Acme',
  description: '...',
  url: 'https://acme.com',    // ← this becomes alternates.canonical + og:url
  // ...
});
```

`defineSeo()` emits the canonical link automatically. The URL also drives `metadataBase`, `og:url`, and (when applicable) the Twitter card URL — one declaration covers all four.

## Per-page canonical override

When a child page should canonicalize to a different URL than its layout:

```tsx filename="app/products/widget/page.tsx"
import { defineSeo } from '@answerable/metadata';

export const metadata = defineSeo({
  title: 'Widget',
  description: '...',
  url: 'https://acme.com/products/widget',
});
```

Each page gets its own canonical. Pagination, tagged views, and UTM-laden share links all point back to the canonical.

## Manual way (without `defineSeo`)

If you're not using `@answerable/metadata`, the Next.js shorthand:

```tsx filename="app/layout.tsx"
import type { Metadata } from 'next';

export const metadata: Metadata = {
  metadataBase: new URL('https://acme.com'),
  alternates: {
    canonical: '/',
  },
};
```

Note `metadataBase` — without it, Next.js emits warnings about resolving relative canonical paths.

## Common mistakes

- **Canonical pointing at a different URL than the page itself.** This is fine when intentional (e.g., a paginated `/blog?page=3` canonicalizes to `/blog`), but easy to do accidentally by pasting the layout's URL into a child page.
- **Relative canonical paths.** Audit check **A4** warns on relative canonicals (still valid, but crawled inconsistently). Use absolute https URLs.
- **Trailing-slash mismatch.** Pick one (with or without `/`) and stick with it across canonical, sitemap, OG URL, and the actual route. Mixing them is the #1 way to split ranking signal.
